Compile Code

Allows to run a compiler on snippets at build time and inject the generated output in the final web site.

Usage​

After configuring the plugin for your language snippets, the plugin will automatically run your tool and insert the console output and error in the page.

```z3wasm
(declare-const a Int)
(declare-fun f (Int Bool) Int)
(assert (= a 10))
(assert (= (f a true) 100))
(check-sat)
```

Configuration​

This configuration allows to extract code snippets and run them through your tool at build time, then insert the output in the generated web site. As long as your tool can run in a GitHub Action container, this feature allows to easily inline the output of your tool.

./docusaurus.config.js
...
const config = configure({ ... }, {
    compileCode: {
        langs: [{
            lang: "...",
            ...
        }, ...]
    }
})

Supported tools​

  • Tools published on npm
  • Node.JS JavaScript script
  • HTML/JavaScript page
  • Custom tool

Tool configuration flags​

meta​

An additional meta filter for selecting snippet. If this field is preset, snippet without the meta (case sensitive) will be ignored.

./docusaurus.config.js
    ...
    {
        lang: "ts",
        meta: "check",
    },
```ts check
let x = 0;
```

inputLang​

After processing the source, replace the lang value with inputLang, so that it gets syntax colored with that language.

Set to null to remove the original code from the generate output.

inputFiles​

Specify a set of files to be written in the working folder before running the tool.

./docusaurus.config.js
    ...
    {
        lang: "ts",
        nodeBin: "tsc",
        inputFiles: {
            "tsconfig.json": {
                "compilerOptions": {
                    ...
                },
            }
        }
    },

outputLang​

By default, the console output is rendered using lang value. With outputLang, you can configure how the output show be rendered.

Set to null to remove the console output from the generate markdown.

outputMeta​

outputMeta is an optional string added to the meta of the generated code section.

errorLang, errorMeta​

By default, the error output is rendered using text value. With errorLang, you can configure how the error show be rendered.

Set to null to remove the error output from the generate markdown.

errorMeta​

errorMeta is an optional string added to the meta of the generated code section.

outputFiles​

If the tool produces additional files, they can be listed in the configuration so that the files gets imported in the output.

  • Images (svg, png, jpeg) will automatically be imported as images in the mardown.
  • Other files are injected as tabbed code section.
./docusaurus.config.js
    ...
    {
        lang: "svgo",
        extension: "svg",
        inputLang: "markup",
        command: "./langs/svgo.js",
        outputFiles: [
            {
                name: "output.svg",
                title: "Optimized SVG",
            },
            {
                name: "output.svg.txt",
                lang: "markup",
                title: "optimized.svg",
            },
        ],
    },

hashFiles​

A set of files to be included in the hash computation for the cache.

./docusaurus.config.js
    ...
    {
        lang: "svgo",
        hashFiles: [
            "package.json",
        ],
    },

Meta flags​

skip​

Completely ignore snippet by adding skip in the meta.

```echo skip
just ignore me
```
http://localhost:3000
just ignore me

ignoreErrors​

Don't fail the rendering step if errors occur.

```ts ignoreErrors
console.lllllog("yup, there's a typo");
```
http://localhost:3000
console.lllllog("yup, there's a typo");
Output
input.ts(1,9): error TS2339: Property 'lllllog' does not exist on type 'Console'.

Outputs and cache​

The extracted sources and resulting outputs are stored by default in the ./.docusaurus/docusaurus-remark-plugin-compile-code/ folder.

Make sure to clear this folder if you encounter caching issues.
